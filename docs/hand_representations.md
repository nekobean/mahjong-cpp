# 手牌表現の種類

このページでは、麻雀プログラムで使う手牌表現を整理します。目的に応じて表現を使い分けることで、処理の意図が読みやすくなります。

## 表現の一覧

| 表現                   | 説明                                     |
| ---------------------- | ---------------------------------------- |
| ビット表現方式         | 牌種ごとの枚数をビット列に詰めた圧縮表現 |
| 配列 (牌種ごとの枚数)  | 34 種類の牌それぞれの枚数を持つ配列      |
| 配列 (牌1枚ごとの配列) | 手牌に含まれる牌を 1 枚ずつ並べた配列    |

## ビット表現方式

牌種ごとの枚数を整数のビット列に詰める方式です。たとえば 1 牌種は最大4枚であるため、3 bit を使えば、各牌の枚数 `0..4` を表現できます。

```txt
010 000 001 000 011 000 000 000 001
 1m  2m  3m  4m  5m  6m  7m  8m  9m
```

この例では、左から `1m=2枚, 2m=0枚, 3m=1枚, 4m=0枚, 5m=3枚, ..., 9m=1枚` を表します。

```cpp
using PackedTileCounts = std::uint32_t;
```

複数の色をまとめる場合は、萬子・筒子・索子・字牌ごとに分けると扱いやすくなります。

```cpp
struct {
    PackedTileCounts manzu;
    PackedTileCounts pinzu;
    PackedTileCounts souzu;
    PackedTileCounts honors;
};
```

この方式は、特定の牌が含まれているかをマスクで一度に確認できるため、役判定などに向いています。一方で、通常の手牌操作には読みづらいため、内部的な高速判定用の表現として扱うのが自然です。

```cpp
// 混老頭かどうかの判定
const PackedTileCounts non_terminals_mask = 0b000'111'111'111'111'111'111'111'000;
const bool has_no_non_terminals =
    ((manzu | pinzu | souzu) & non_terminals_mask) == 0;
```

## 配列 (牌種ごとの枚数)

34 種類の牌それぞれについて、手牌に何枚あるかを持つ方式です。

```cpp
using TileCountArray = std::array<int, 34>;
```

たとえば `counts[Tile::Manzu1] == 2` なら、手牌に `1m` が 2 枚あることを表します。

この方式は、麻雀プログラムで最も扱いやすい基本表現です。枚数チェックなど、牌種ごとの枚数を直接見たい処理に向いています。

## 配列 (牌1枚ごとの配列)

手牌に含まれる牌を 1 枚ずつ並べる方式です。

```cpp
using TileSequence = std::vector<int>;
```

たとえば次のように、同じ牌は枚数分だけ要素として持ちます。

```cpp
std::vector<int> tiles = {
    Tile::Manzu1,
    Tile::Manzu1,
    Tile::Manzu3,
    Tile::Pinzu5,
};
```

この方式は、人間が入力した牌列や、表示・ソート・列挙に向いています。一方で、ある牌が何枚あるかを何度も調べる処理では、毎回数える必要があるため、配列 (牌種ごとの枚数) へ変換してから扱う方が分かりやすくなります。

## 速度面での比較

手牌操作の主なユースケースごとに、速度面での向き不向きを整理します。

### ある牌が存在するか

- ビット表現方式: ◎ 対象牌のビット幅を参照して判定できる
- 配列 (牌種ごとの枚数): ◎ `counts[tile] > 0` で定数時間
- 配列 (牌1枚ごとの配列): △ 線形探索が必要

### ある牌が含まれる数を数える

- ビット表現方式: ◎ 対象牌のビット幅を取り出して取得できる
- 配列 (牌種ごとの枚数): ◎ `counts[tile]` で定数時間
- 配列 (牌1枚ごとの配列): △ 全体走査が必要

### 牌を1枚追加する

- ビット表現方式: ○ bit 更新は定数時間だが操作がやや重い
- 配列 (牌種ごとの枚数): ◎ `++counts[tile]` で定数時間
- 配列 (牌1枚ごとの配列): ◎ 末尾追加なら定数時間

### 牌を1枚削除する

- ビット表現方式: ○ bit 更新は定数時間だが操作がやや重い
- 配列 (牌種ごとの枚数): ◎ `--counts[tile]` で定数時間
- 配列 (牌1枚ごとの配列): △ 対象牌の探索と削除が必要

### 手牌の総枚数を数える

- ビット表現方式: △ 各牌種の枚数を取り出して合計する
- 配列 (牌種ごとの枚数): ○ 配列の要素の合計
- 配列 (牌1枚ごとの配列): ◎ `tiles.size()` で定数時間

### 特定範囲の牌を判定する

- ビット表現方式: ◎ マスク演算でまとめて判定できる
- 配列 (牌種ごとの枚数): ○ 対象範囲の走査が必要
- 配列 (牌1枚ごとの配列): △ 全体走査が必要

### 同じ手牌か比較する

- ビット表現方式: ◎ 整数値の比較で速い
- 配列 (牌種ごとの枚数): ○ 34 要素の比較
- 配列 (牌1枚ごとの配列): △ 並び順のソートが必要
