# 天鳳 MJLOG 仕様

## はじめに

本資料は、オンライン麻雀「天鳳」が出力する牌譜ファイル `mjlog` の形式と、その読み解き方をまとめたものである。点数計算の検証やテストデータの生成にあたり、牌譜から局の進行・手牌・和了内容を復元する際の参照資料として用いることを想定している。

## ファイル構造

`mjlog` は XML のタグ列で構成される。1つのファイルが東風戦または東南戦1試合に対応し、配信時には gzip で圧縮されていることがある。

ファイル全体はおおむね次の構造になる。

```xml
<mjloggm ...>
  <SHUFFLE .../>
  <GO .../>
  <UN .../>
  <TAIKYOKU .../>
  <INIT .../>
  ...
  <AGARI .../>
  <INIT .../>
  ...
  <RYUUKYOKU .../>
</mjloggm>
```

冒頭の `SHUFFLE`・`GO`・`UN`・`TAIKYOKU` で試合全体の情報を与えたあと、局ごとのイベント列が
続く。1局は `INIT` で始まり、`AGARI`（和了）または `RYUUKYOKU`（流局）で終わる。
ダブロンのように1局で複数人が和了した場合は、`AGARI` が連続して出現する。

タグは入れ子を持たず、時系列順に直列で並ぶ。したがって解析側は、タグを順に読みながら
ツモ・打牌・副露などのイベントを適用し、各時点の盤面を再構築すればよい。

## 副露値

副露やカン、抜きドラは `N` タグの `m` 属性に集約され、`m` は 16 bit のビットフィールドとして
表現される。下位2 bit が鳴き元、残りが牌情報を表す。

| bit          | マスク   | 内容   |
| ------------ | -------- | ------ |
| `0-1 bit目`  | `0x0003` | 鳴き元 |
| `2-15 bit目` | `0xFFFC` | 牌情報 |

牌情報の解釈は副露種別によって変わる。まず副露種別を判定し、そのあと種別ごとの形式で復号する。

### 判定手順

副露種別の判定は次のように書ける。

```txt
from = m & 0x0003

is_chi = (m & 0x0004) != 0
is_pon = (m & 0x000C) == 0x0008
is_added_kan = (m & 0x0014) == 0x0010
is_nuki = (m & 0x003C) == 0x0020

if is_chi:
    decode as chi
else if is_pon:
    decode as pon
else if is_added_kan:
    decode as added kan
else if is_nuki:
    decode as nuki
else:
    decode as open kan or closed kan
```

### 鳴き元

下位2 bit は、鳴いたプレイヤーから見た相対方向を表す。

| 値  | 意味           |
| --- | -------------- |
| `0` | 抜きドラ・暗槓 |
| `1` | 下家           |
| `2` | 対面           |
| `3` | 上家           |

### 種別ごとの復号

以下に、副露種別ごとの `2-15 bit目` の解釈を示す。

### チー

チーは `bit 2` が立っていることで判定する。

```txt
is_chi = (m & 0x0004) != 0
```

| bit           | マスク   | 内容                                           |
| ------------- | -------- | ---------------------------------------------- |
| `0-1 bit目`   | `0x0003` | 誰から鳴いたか                                 |
| `2 bit目`     | `0x0004` | チーフラグ                                     |
| `3-4 bit目`   | `0x0018` | 1枚目が同種4枚のどれかを表すインデックス (0~3) |
| `5-6 bit目`   | `0x0060` | 2枚目が同種4枚のどれかを表すインデックス (0~3) |
| `7-8 bit目`   | `0x0180` | 3枚目が同種4枚のどれかを表すインデックス (0~3) |
| `10-15 bit目` | `0xFC00` | 順子パターン                                   |

```txt
from = m & 0x0003
index0 = (m & 0x0018) >> 3
index1 = (m & 0x0060) >> 5
index2 = (m & 0x0180) >> 7
t = (m & 0xFC00) >> 10

called_index = t % 3
min_pai = t / 3
base136 = ((min_pai / 7) * 9 + (min_pai % 7)) * 4

tile0 = base136 + index0
tile1 = base136 + 4 + index1
tile2 = base136 + 8 + index2
called_tile = [tile0, tile1, tile2][called_index]
```

`min_pai` は順子の最小牌を表し、`0..6 = 1m..7m`、`7..13 = 1p..7p`、`14..20 = 1s..7s` に
対応する。

### ポン

ポンは `bit 2` が立っておらず、`bit 3` が立っていることで判定する。

```txt
is_pon = (m & 0x000C) == 0x0008
```

| bit          | マスク   | 内容                                                |
| ------------ | -------- | --------------------------------------------------- |
| `0-1 bit目`  | `0x0003` | 誰から鳴いたか                                      |
| `2 bit目`    | `0x0004` | 0                                                   |
| `3 bit目`    | `0x0008` | 1                                                   |
| `5-6 bit目`  | `0x0060` | 同種4枚のどれが含まれないかを表すインデックス (0~3) |
| `9-15 bit目` | `0xFE00` | 刻子パターン                                        |

```txt
from = m & 0x0003
unused_index = (m & 0x0060) >> 5
t = (m & 0xFE00) >> 9

called_index = t % 3
base136 = (t / 3) * 4

tiles = []
for i in 0..3:
    if i != unused_index:
        tiles.push(base136 + i)

called_tile = tiles[called_index]
```

`base136..base136+3` の4枚から、`unused_index` に対応する1枚を除いた3枚が構成牌になる。

### 加槓

加槓は `bit 2` が立っておらず、`bit 4` が立っていることで判定する。

```txt
is_added_kan = (m & 0x0014) == 0x0010
```

| bit          | マスク   | 内容                                              |
| ------------ | -------- | ------------------------------------------------- |
| `0-1 bit目`  | `0x0003` | 誰から鳴いたか                                    |
| `2 bit目`    | `0x0004` | 0                                                 |
| `4 bit目`    | `0x0010` | 1                                                 |
| `5-6 bit目`  | `0x0060` | 後から加えた牌が同種4枚のどれかを表すインデックス |
| `9-15 bit目` | `0xFE00` | 刻子パターン                                      |

```txt
from = m & 0x0003
added_index = (m & 0x0060) >> 5
t = (m & 0xFE00) >> 9

base136 = (t / 3) * 4
added_tile = base136 + added_index
tiles = [base136, base136 + 1, base136 + 2, base136 + 3]
```

`bit 9-15` の刻子パターンは、元のポンと同じ形式で保持される。
`added_tile` が、加槓で新たに加えた1枚を表す。

### 明槓・暗槓

`bit 2-5` がすべて 0 の場合は、明槓または暗槓として扱う。両者は鳴き元（`bit 0-1`）で見分ける。

```txt
is_kan = (m & 0x003C) == 0
is_closed_kan = (m & 0x0003) == 0
is_open_kan = (m & 0x0003) != 0
```

| bit          | マスク   | 内容                                          |
| ------------ | -------- | --------------------------------------------- |
| `0-1 bit目`  | `0x0003` | 誰から鳴いたか。暗槓なら `0`、明槓なら `1..3` |
| `2-5 bit目`  | `0x003C` | 0                                             |
| `8-15 bit目` | `0xFF00` | 槓子パターン                                  |

```txt
from = m & 0x0003
raw = (m & 0xFF00) >> 8

base136 = raw - (raw % 4)
called_tile = raw
tiles = [base136, base136 + 1, base136 + 2, base136 + 3]
```

暗槓では `from == 0` となり、鳴かれた牌という概念がないため、実装上は `called_tile` を
使わないことが多い。明槓では `raw` がそのまま鳴いた牌を表す。

### 抜きドラ

抜きドラ（三麻の北抜き）は、`bit 2-4` がすべて 0 で `bit 5` が 1 のときに判定する。

```txt
is_nuki = (m & 0x003C) == 0x0020
```

| bit          | マスク   | 内容             |
| ------------ | -------- | ---------------- |
| `0-1 bit目`  | `0x0003` | 0                |
| `2-4 bit目`  | `0x001C` | 0                |
| `5 bit目`    | `0x0020` | 1                |
| `8-15 bit目` | `0xFF00` | 抜いた牌の牌番号 |

```txt
raw = (m & 0xFF00) >> 8
base136 = raw - (raw % 4)
nuki_tile = raw
```

通常は北を抜くため、`nuki_tile` は `120..123` のいずれかになる。

## タグ

### mjloggm

牌譜全体を包むルートタグ。

| 属性  | 内容       |
| ----- | ---------- |
| `ver` | バージョン |

例:

```xml
<mjloggm ver="2.3">
  ...
</mjloggm>
```

`mjloggm` の内側に `SHUFFLE`・`GO`・`UN`・`TAIKYOKU` と、各局のイベント列が時系列順に並ぶ。

### SHUFFLE

牌山の生成情報を表すタグ。牌山の再現や検証に用いる。

| 属性   | 内容                           |
| ------ | ------------------------------ |
| `seed` | 乱数シード文字列               |
| `ref`  | 山生成結果に対応する参照文字列 |

例:

```xml
<SHUFFLE seed="mt19937ar,..." ref="..."/>
```

### GO

卓種別・ルール情報を表すタグ。`type` のビットの意味は「卓種別」を参照。

| 属性    | 内容                      |
| ------- | ------------------------- |
| `type`  | 卓種別を表す整数          |
| `lobby` | ロビー ID。段位戦では `0` |

例:

```xml
<GO type="169" lobby="0"/>
```

### UN

プレイヤー情報を表すタグ。

| 属性     | 内容                                                     |
| -------- | -------------------------------------------------------- |
| `n0..n3` | プレイヤー名。URL エンコードされている                   |
| `dan`    | 各プレイヤーの段位 ID                                    |
| `rate`   | 各プレイヤーのレート                                     |
| `sx`     | 各プレイヤーの性別 (M: 男性、F: 女性、C: コンピューター) |

例:

```xml
<UN n0="%E7%8E%A9%E5%AE%B6A"
    n1="%E7%8E%A9%E5%AE%B6B"
    n2="%E7%8E%A9%E5%AE%B6C"
    n3="%E7%8E%A9%E5%AE%B6D"
    dan="16,16,15,17"
    rate="2000,1988,2033,2101"
    sx="M,M,F,M"/>
```

`UN` は対局開始時だけでなく、回線落ちしたプレイヤーの再接続時にも出現する。
三麻では `n3=""` となり、`dan`・`rate`・`sx` の4人目はダミー値になる。

### TAIKYOKU

対局開始情報を表すタグ。

| 属性  | 内容                    |
| ----- | ----------------------- |
| `oya` | 東1局の起家のプレイヤー |

例:

```xml
<TAIKYOKU oya="1"/>
```

ここでの `oya` は対局全体の起家を指す。各局の親は `INIT` の `oya` を使う。

### INIT

局開始情報を表すタグ。局の初期状態（持ち点・親・配牌）を与える。

| 属性         | 内容                                    |
| ------------ | --------------------------------------- |
| `seed`       | 局情報                                  |
| `ten`        | 各プレイヤーの持ち点。下2桁は省略される |
| `oya`        | 親のプレイヤー番号                      |
| `hai0..hai3` | 各プレイヤーの配牌                      |

例:

```
<INIT seed="0,0,0,3,2,92"
      ten="250,250,250,250"
      oya="0"
      hai0="30,114,108,31,78,107,25,23,2,14,122,44,49"
      hai1="..."
      hai2="..."
      hai3="..."/>
```

`seed` はカンマ区切りの6つの整数で、次の意味を持つ。

| 位置 | 内容                    |
| ---- | ----------------------- |
| `0`  | 局番号                  |
| `1`  | 積み棒                  |
| `2`  | 供託リーチ棒            |
| `3`  | 1つ目のサイコロの目 - 1 |
| `4`  | 2つ目のサイコロの目 - 1 |
| `5`  | 初期ドラ表示牌          |

`hai0..hai3` はカンマ区切りの13枚の牌番号で配牌を表す。親も13枚で記録される。
三麻では、4人目のプレイヤーの配牌は `hai3=""` となる。

### T{N}/U{N}/V{N}/W{N}

ツモを表すタグ。タグ名そのものにプレイヤーと牌番号が含まれる。
プレイヤーとの対応は `T = プレイヤー0`、`U = プレイヤー1`、`V = プレイヤー2`、`W = プレイヤー3`。
たとえば `<T52/>` はプレイヤー0が牌番号52をツモしたことを表す。

### D{N}/E{N}/F{N}/G{N}

打牌を表すタグ。タグ名そのものにプレイヤーと牌番号が含まれる。
プレイヤーとの対応は `D = プレイヤー0`、`E = プレイヤー1`、`F = プレイヤー2`、`G = プレイヤー3`。
たとえば `<F88/>` はプレイヤー2が牌番号88を打牌したことを表す。

### N

副露、または抜きドラを表すタグ。`m` のデコード方法は「副露値」を参照。

| 属性  | 内容     |
| ----- | -------- |
| `who` | 鳴いた人 |
| `m`   | 副露値   |

例:

```xml
<N who="2" m="34314"/>
```

### DORA

槓ドラ表示牌の追加を表すタグ。

| 属性  | 内容       |
| ----- | ---------- |
| `hai` | ドラ表示牌 |

例:

```xml
<DORA hai="117"/>
```

明槓・加槓ではツモ後に、暗槓では副露直後に出現する。

### BYE

回線落ちしたプレイヤーを表すタグ。

| 属性  | 内容               |
| ----- | ------------------ |
| `who` | 離脱したプレイヤー |

例:

```xml
<BYE who="2"/>
```

### REACH

リーチ宣言を表すタグ。

| 属性   | 内容                                          |
| ------ | --------------------------------------------- |
| `who`  | リーチ者                                      |
| `step` | `1` = 宣言時、`2` = 供託確定時                |
| `ten`  | `プレイヤー0の持ち点, ...`。下2桁は省略される |

例:

```xml
<REACH who="0" step="1"/>
<REACH who="0" step="2" ten="240,250,250,250"/>
```

1回の立直につき、宣言時の `step="1"` と供託確定時の `step="2"` が2回に分かれて出る。
リーチ宣言牌で放銃してリーチが不成立になった場合は、`step="2"` が出ない。

### AGARI

和了情報を表すタグ。

| 属性         | 内容                                                         |
| ------------ | ------------------------------------------------------------ |
| `ba`         | `本場,供託リーチ棒`                                          |
| `hai`        | 和了者の手牌。ロン和了ではロン牌も含まれる                   |
| `m`          | 和了者の副露値の一覧。副露がある場合のみ存在                 |
| `machi`      | 和了した牌                                                   |
| `ten`        | `符,点数,点数タイトル`。点数は下2桁を省略                    |
| `yaku`       | `役 ID,翻数,役 ID,翻数, ...`。役満時は空                     |
| `yakuman`    | `役 ID,役 ID, ...`。役満時のみ存在                           |
| `doraHai`    | 表ドラ表示牌の一覧                                           |
| `doraHaiUra` | 裏ドラ表示牌の一覧。リーチ時のみ存在                         |
| `who`        | 和了者                                                       |
| `fromWho`    | 放銃者。ツモの場合は `who == fromWho`                        |
| `sc`         | `プレイヤー0の点棒,プレイヤー0の収支, ...`。下2桁を省略      |
| `owari`      | `プレイヤー0の点棒,プレイヤー0のスコア, ...`。終局時のみ存在 |

補足:

- `yaku`: リーチ和了のときは、裏ドラが付いていなくても裏ドラが記載される。その場合の翻数は `0`。
- `sc`: 変動前の点棒と収支を交互に並べたもの。
- `owari`: スコアにはウマを含む。

#### 包

大三元と大四喜には包（責任払い）がある。
包が発生した場合、得点はツモなら包責者が全額、ロンなら包責者と放銃者が折半で支払う。
積み棒は包責者が支払う。
たとえば大三元の包では、`who == fromWho` のツモ和了であっても、`sc` 上は責任者1人だけが
得点を支払う形になる。

#### ダブロン

ダブロンでは `<AGARI .../>` が連続して出現する。
記載順は、放銃者から見てツモ順が早いプレイヤーの和了から先になる。なお、トリロンは流局扱いの
ため存在しない。
供託リーチ棒はツモ順の早いプレイヤーにだけ入る。たとえば、一人目は `ba="1,1"`、二人目は `ba="1,0"` になる。

### RYUUKYOKU

流局情報を表すタグ。通常流局のほか、途中流局もこのタグで表現される。

| 属性         | 内容                                                         |
| ------------ | ------------------------------------------------------------ |
| `ba`         | `本場,供託リーチ棒`                                          |
| `sc`         | `プレイヤー0の点棒,プレイヤー0の収支, ...`。下2桁を省略      |
| `hai0..hai3` | 聴牌者の手牌。非聴牌者は属性自体がないことがある             |
| `type`       | 流局種別                                                     |
| `owari`      | 終局時のみ存在。`プレイヤー0の点棒,プレイヤー0のスコア, ...` |

`type` に入る主な値は次の通り。

| 値       | 意味     |
| -------- | -------- |
| `yao9`   | 九種九牌 |
| `kaze4`  | 四風連打 |
| `reach4` | 四家立直 |
| `ron3`   | 三家和了 |
| `kan4`   | 四槓散了 |
| `nm`     | 流し満貫 |

補足:

- `sc`: 変動前の点棒と収支を交互に並べたもの。
- `owari`: スコアにはウマを含む。

## ドラ表示牌

`doraHai` と `doraHaiUra` は、ドラそのものではなくドラ表示牌として扱う。表示牌から実際のドラへの
変換は通常のルールに従う。

ただし三麻では萬子2〜8を使わないため、萬子のドラ循環だけは次のように扱う。

| 表示牌 | ドラ |
| ------ | ---- |
| 1m     | 9m   |
| 9m     | 1m   |
